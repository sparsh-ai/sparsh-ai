Model is trained. Now what?

Posted on Sat, Jan 16, 2021 Deployment

Prototype vs. Production

As you can see in the above table, there are 2 columns - prototype and production. If you are working on any proof-of-concept or performing user acceptance testing (UAT), you can use prototype pipeline (i.e. forcus on first colum only, forget about the 2nd one).

Step 0: Model

Scikit, Tensorflow and Pytorch are my favorite frameworks for model building. Sometimes, I need to build a prototype model real-quick (like <30 mins). In this case, I use Teachable machine for some use cases like pose estimation.

Step 1: App

Prototype: I use Gradio a lot for serving my model. Gradio wraps my model as an API in less than 5 mins with minimal coding. Streamlit is my 2nd favorite after Gradio. Streamlit takes a little bit of time (like 20-30 mins) for wrapping but gives me a lot more functionality and flexibility. It is super cool. I recently noticed that the 'H2O Wave' is making some waves. I think it can compete with Gradio and Streamlit. I am excited to try it anytime soon.

Production: FastAPI is actually fast for serving models as an API. FlaskAPI comes next. For the dashboard, my favorite is Plotly Dash. It is very powerful and flexible. And again, Dash is super cool. Tableau is my 2nd favorite and lots of industrial demand is in tableau. PowerBI is also a good option. I think PowerBI and Google Data Studio are very user friendly. The entire dashboard (from the functionality perspective) can be created in 30-60 mins.

Step 2: Container

Prototype: For some use cases (privacy reasons), I use the local machine. Otherwise, I prefer to use Colab as my container for deploying my model.

Production: Docker is my favorite container for shipping out ML models.

Step 3: Hosting (on cloud)

Prototype: Again colab. If no privacy restrictions, I prefer colab for hosting. It can host for half a day in a given session. So a good choice for me. Heroku is my favorite if multi-day hosting is required for testing the prototypes.

Production: EBS is nice. I use it a lot for serving the models as REST APIs. Kubernetes is super powerful for the orchestration of multiple APIs.

Step 4: Hosting (on edge)

Prototype: ml5.js is super cool and powerful. I can directly serve my ML models on the browser using ml5.js

Production: TF.js and TFLite are my favorites for serving on the browser and on mobile respectively. I never tried ml5.js in production but I think It can be used in production also because it is built on top of TF.js.
