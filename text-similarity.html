About

Text Similarity,_,

Text similarity has to determine how ‘close’ two pieces of text are both in surface closeness (lexical similarity) and meaning (semantic similarity)

Applications - Duplicate document detection, text clustering, product recommendations

Tools - Sentence Transformer Library, Universal Sentence Encoder Model (TFHub), Scikit-learn

Models

  1. BERT: Use transfer learning to fine-tune a BERT encoder. This encoder will work as a feature extractor. e.g. the most common version of BERT converts any given text into a numeric vector of length 768 (this vector is also known as a contextual embedding).
  2. Bag-of-words: Extract features using models like TF-IDF, CountVectorizer.
  3. DeepRank: DeepRank: A New Deep Architecture for Relevance Ranking in Information Retrieval. arXiv, 2017.
  4. FAISS: Billion-scale similarity search with GPUs. arXiv, 2017. Faiss is a library for efficient similarity search and clustering of dense vectors.
  5. Similarity Measures: L1 (Manhatten distance), L2 (Euclidean distance), Hinge Loss for Triplets.

Use Cases

  1. Semantic Relation Estimation - To maintain a level of coherence and similarity among various letters and speeches, a model was built that will help in assessing this document similarity. In approach 1, TF-IDF with Latent semantic indexing was used to extract features and cosine similarity as the distance metric. In approach 2, BERT with PCA was used for feature extraction and 3 distance measures — L1, L2, and cosine, for similarity calculation. Check out this notion.
  2. Finding Hardware Parts in Warehouse - There are millions of hardware items (e.g. 0.5mm steel wire grade q195) in the warehouse and customer generally asks for items in natural language (e.g. grade195 steel wire with 0.5mm thickness). A text similarity system was built using an ensemble of 3 Bag-of-words based Count vectorizer model with different types of tokenization process and n-gram range. Check out this notion.
  3. Image + Text Similarity - Use the textual details and images of products, find the exact similar product among different groups. Around 35 GB of retail product images was scraped and used to build the system. Checkout the notion here.
  4. Text Recommendation - For the given BRM text, recommend top-5 GAO text. We used universal sentence encoder to encode the text and calculated cosine similarity within group. Then an item-based recommender model was used to find most suitable top-K candidates in GAO based on the interaction history. Check out this notion.
